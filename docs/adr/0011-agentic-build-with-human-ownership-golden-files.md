# Decision 0011: Agentic implementation, human ownership of golden files

**Status:**      Accepted
**Date:**        2026-07-13

## Context

Available budget: 8–16 hrs/wk × ~4.5 weeks until 12 August = 36–72 hrs
total. Manual building doesn't fit; fully unreviewed generation clashes
with a product whose correctness is the value, with a public AGPL repo
(0008), and with the maintainer role toward community.

## Decision

The agent (Claude Code) writes ~80% of the code mass (scaffolding,
auth, billing, CRUD, imports, scan, dashboard); golden files and
acceptance tests are authored exclusively by the human from official PRO
specs, and engine plus SchemeAdapter code is reviewed line by line; CI
with golden files is the gatekeeper.

## Considerations

**Pro:** only feasible route within the hour budget; the architecture
(pure engine, adapters, golden files — 0002/0003, N-01) is precisely
built to let tests drive; expensive hours go to research, review, and
GTM instead of typing forms.
**Con:** requires discipline not to skip core review under time pressure
— hallucinated declaration formats are fatal, so golden files never
generated by the agent.

Rejected: all manual (misses the deadline window); fully agentic
without review (correctness, liability, and maintainer risk).

## Consequences

- Hour allocation: ±40% regulation + golden files, ±30% core review,
  ±30% validation/GTM/deploy
- Work rhythm per module: spec/tests → agent builds → review → merge
- The free scan may be fully agentic (checklist, no calculation)
